# Style and conventions
- Language: TypeScript with strict compiler settings (`strict`, `noImplicitAny`, `noImplicitOverride`, `noImplicitReturns`, `noImplicitThis`).
- Modules: ESM (`"type": "module"`, `moduleResolution: bundler`).
- Formatting style seen in source:
  - Tabs for indentation in TS source.
  - Semicolons used consistently.
  - Prefer explicit typing for public APIs and complex structures.
  - Use `type`-only imports where appropriate (`import { type X }`).
- Naming:
  - Classes: PascalCase (`WitnessCore`, `PuzzleGenerator`).
  - Methods/variables: camelCase.
  - Constants/enums: UPPER_SNAKE for constants, PascalCase enum names + enum members.
- Documentation/comments:
  - Japanese JSDoc comments are common for classes/methods/types.
  - Keep comments concise and implementation-oriented.
- Error handling:
  - Fallback/retry logic used in generation paths.
  - Build scripts fail fast with process exit on command failure.
- Compatibility note:
  - Current development context indicates beta orientation; backward compatibility is not prioritized unless explicitly requested.
